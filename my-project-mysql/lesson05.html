MySQLデータ型と制約の理解

基本データ型

MySQLにはいくつかの基本的なデータ型があります。

/* 基本データ型 */
- INT: 整数値
- VARCHAR(n): 可変長文字列
- DATE: 日付
- FLOAT: 浮動小数点数

制約の種類

制約は、テーブルのデータの整合性を確保するために使用されます。

/* 制約の種類 */
- PRIMARY KEY: 一意の識別子
- FOREIGN KEY: 他のテーブルとの関係を示す
- NOT NULL: NULL値を許可しない
- UNIQUE: 一意の値を要求する

VARCHARの使用例

VARCHAR型は、文字列データを格納するためのデータ型です。

/* VARCHARの使用例 */
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE
);

INTの使用例

INT型は、整数値を格納するためのデータ型です。

/* INTの使用例 */
CREATE TABLE orders (
    order_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    amount FLOAT
);

DATE型の使用例

DATE型は、日付を格納するためのデータ型です。

/* DATE型の使用例 */
CREATE TABLE events (
    event_id INT AUTO_INCREMENT PRIMARY KEY,
    event_date DATE NOT NULL
);

FOREIGN KEYの使用例

FOREIGN KEY制約を使用して、他のテーブルとの関係を定義します。

/* FOREIGN KEYの使用例 */
CREATE TABLE orders (
    order_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

まとめ

データ型と制約を理解することで、データベース設計がより効果的になります。
