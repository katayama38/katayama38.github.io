MySQLインデックスの理解

インデックスの概要

インデックスは、データベースのパフォーマンスを向上させるために使用されます。

/* インデックスの概要 */
- データの検索速度を向上させる
- 特定のカラムに対するアクセスを最適化

インデックスの種類

主なインデックスの種類には、以下があります。

/* インデックスの種類 */
- プライマリインデックス: 主キーに自動的に作成される
- ユニークインデックス: 重複を許さないインデックス
- 通常インデックス: 一般的なインデックス
- フルテキストインデックス: テキスト検索用のインデックス

インデックスの作成例

インデックスを作成するためのSQL文の例です。

/* インデックスの作成 */
CREATE INDEX idx_name ON users (name);

プライマリインデックスの例

プライマリインデックスは、主キーに自動的に作成されます。

/* プライマリインデックスの例 */
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100)
);

ユニークインデックスの例

ユニークインデックスは、指定したカラムに重複がないことを保証します。

/* ユニークインデックスの例 */
CREATE TABLE products (
    product_id INT AUTO_INCREMENT PRIMARY KEY,
    product_code VARCHAR(50) UNIQUE
);

フルテキストインデックスの使用例

フルテキストインデックスは、テキスト検索を最適化します。

/* フルテキストインデックスの使用例 */
CREATE TABLE articles (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255),
    content TEXT,
    FULLTEXT (title, content)
);

インデックスの削除

不要になったインデックスを削除する方法です。

/* インデックスの削除 */
DROP INDEX idx_name ON users;

まとめ

インデックスを適切に利用することで、データベースのパフォーマンスが向上します。
