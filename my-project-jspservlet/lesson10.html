エラーハンドリングの理解

エラーハンドリングの概要

エラーハンドリングは、プログラムの実行中に発生するエラーを処理するための手法です。

例外とは

例外は、プログラムの実行中に発生する異常な状況を表します。Javaでは、Throwable クラスから派生するオブジェクトとして表現されます。

例外の捕捉

例外を捕捉するには、try-catch 文を使用します。

try {
    // 例外が発生する可能性のあるコード
} catch (Exception e) {
    // 例外が発生した場合の処理
}

複数の例外の捕捉

異なる種類の例外を捕捉するには、複数の catch ブロックを使用します。

try {
    // コード
} catch (IOException e) {
    // IO例外の処理
} catch (SQLException e) {
    // SQL例外の処理
}

例外の再スロー

捕捉した例外を再スローするには、throw 文を使用します。

catch (Exception e) {
    throw e; // 再スロー
}

カスタム例外の作成

独自の例外クラスを作成するには、Exception クラスを継承します。

public class MyException extends Exception {
    public MyException(String message) {
        super(message);
    }
}

finallyブロックの使用

例外の有無にかかわらず、必ず実行したいコードは finally ブロックに書きます。

try {
    // コード
} catch (Exception e) {
    // 例外処理
} finally {
    // 常に実行されるコード
}

まとめ

エラーハンドリングを適切に行うことで、プログラムの信頼性が向上し、予期しない状況に対処できるようになります。
