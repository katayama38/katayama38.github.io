Java Servletのライフサイクルの理解

Servletのライフサイクルの概要

Servletのライフサイクルは、初期化、リクエスト処理、破棄の3つの主要なフェーズで構成されています。

初期化フェーズ

Servletが最初に呼び出されるとき、init()メソッドが呼び出され、リソースの初期化を行います。

/* 初期化メソッドの例 */
public class InitExampleServlet extends HttpServlet {
    public void init() throws ServletException {
        // リソースの初期化処理
        System.out.println("Servletが初期化されました。");
    }
}

リクエスト処理フェーズ

クライアントからのリクエストを受け取ると、doGet()またはdoPost()メソッドが呼び出されます。

/* リクエスト処理メソッドの例 */
protected void doGet(HttpServletRequest request, HttpServletResponse response)
        throws ServletException, IOException {
    response.setContentType("text/html");
    PrintWriter out = response.getWriter();
    out.println("<h1>リクエストが処理されました。</h1>");
}

破棄フェーズ

Servletが破棄される前に、destroy()メソッドが呼ばれ、リソースの解放などの後処理を行います。

/* 破棄メソッドの例 */
public class DestroyExampleServlet extends HttpServlet {
    public void destroy() {
        // リソースの解放処理
        System.out.println("Servletが破棄されました。");
    }
}

初期化時の設定

初期化時に設定する必要があるパラメータは、web.xmlで指定できます。


<servlet>
    <servlet-name>ExampleServlet</servlet-name>
    <servlet-class>com.example.ExampleServlet</servlet-class>
    <init-param>
        <param-name>paramName</param-name>
        <param-value>paramValue</param-value>
    </init-param>
</servlet>
