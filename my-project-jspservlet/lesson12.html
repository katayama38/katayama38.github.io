リスナーの理解

リスナーの概要

リスナーは、特定のイベント（リクエストの受信、セッションの生成など）を監視し、処理するためのオブジェクトです。

リスナーの実装

リスナーを実装するには、javax.servlet.ServletContextListener などのインターフェースを実装します。

import javax.servlet.*;
import javax.servlet.annotation.WebListener;

@WebListener
public class MyListener implements ServletContextListener {
    public void contextInitialized(ServletContextEvent sce) {
        // コンテキスト初期化処理
    }

    public void contextDestroyed(ServletContextEvent sce) {
        // コンテキスト終了処理
    }
}

リスナーの登録

リスナーは、アノテーションを使用して自動的に登録されます。web.xml への登録は必要ありません。

リクエストリスナーの実装

リクエストイベントを監視するためには、HttpSessionListenerを使用します。

import javax.servlet.http.*;

public class MySessionListener implements HttpSessionListener {
    public void sessionCreated(HttpSessionEvent se) {
        // セッション作成時の処理
    }

    public void sessionDestroyed(HttpSessionEvent se) {
        // セッション破棄時の処理
    }
}

リクエスト属性の監視

リクエスト属性の変更を監視するには、ServletRequestAttributeListenerを実装します。

import javax.servlet.ServletRequestAttributeEvent;
import javax.servlet.ServletRequestAttributeListener;

public class MyRequestAttributeListener implements ServletRequestAttributeListener {
    public void attributeAdded(ServletRequestAttributeEvent srae) {
        // 属性追加時の処理
    }

    public void attributeRemoved(ServletRequestAttributeEvent srae) {
        // 属性削除時の処理
    }

    public void attributeReplaced(ServletRequestAttributeEvent srae) {
        // 属性変更時の処理
    }
}

リスナーのライフサイクル

リスナーは、アプリケーションの開始時に初期化され、終了時にクリーンアップが行われます。

まとめ

リスナーを利用することで、特定のイベントに対する処理を柔軟に追加できます。
