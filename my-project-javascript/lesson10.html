JavaScriptのモジュール化の理解

モジュール化の定義

モジュール化とは、プログラムを小さな独立した単位（モジュール）に分割することで、管理しやすく、再利用可能にする手法です。

/* モジュール化の定義の例 */
console.log("モジュール化の例");

モジュールの作成

JavaScriptでは、ES6以降のモジュールを使って、別ファイルに定義した関数や変数を利用できます。

/* math.js */
export function add(x, y) {
    return x + y;
}

export function subtract(x, y) {
    return x - y;
}

モジュールのインポート

作成したモジュールは、import文を使って他のファイルで利用できます。

/* main.js */
import { add, subtract } from './math.js';

console.log(add(5, 3)); // 8
console.log(subtract(5, 3)); // 2

デフォルトエクスポートの使用

モジュールは、デフォルトエクスポートを使って一つの値をエクスポートすることもできます。

/* calculator.js */
export default function multiply(x, y) {
    return x * y;
}

デフォルトエクスポートのインポート

デフォルトエクスポートされた関数は、任意の名前でインポートできます。

/* main.js */
import multiply from './calculator.js';

console.log(multiply(4, 5)); // 20

モジュールの利点

モジュールを使用することで、コードの再利用性が高まり、メンテナンスが容易になります。

/* モジュールの利点の例 */
console.log("モジュール化による利点");

まとめ

JavaScriptのモジュール化を理解することで、大規模なアプリケーションでも管理が容易になります。
