JavaScriptのエラーハンドリングの理解

エラーハンドリングの定義

エラーハンドリングとは、プログラムの実行中に発生するエラーを処理する方法です。

/* エラーハンドリングの定義の例 */
try {
    // エラーが発生する可能性のあるコード
    let result = riskyFunction();
} catch (error) {
    console.error("エラーが発生しました:", error);
}

try-catch文の基本

try-catch文を使うことで、エラーを捕捉し、適切に処理できます。

/* try-catch文の例 */
function riskyFunction() {
    throw new Error("何かがうまくいかなかった");
}

try {
    riskyFunction();
} catch (error) {
    console.error("キャッチしたエラー:", error.message);
}

finallyブロックの使用

finallyブロックを使用すると、エラーの有無にかかわらず、特定の処理を実行できます。

/* finallyブロックの例 */
try {
    console.log("処理中...");
    throw new Error("エラー発生");
} catch (error) {
    console.error("エラー:", error.message);
} finally {
    console.log("finallyブロックが実行されました");
}

カスタムエラーの作成

独自のエラーを作成して、特定の状況でエラーをスローできます。

/* カスタムエラーの例 */
class CustomError extends Error {
    constructor(message) {
        super(message);
        this.name = "CustomError";
    }
}

function testCustomError() {
    throw new CustomError("カスタムエラーが発生しました");
}

try {
    testCustomError();
} catch (error) {
    console.error(error.name + ": " + error.message);
}

Promiseにおけるエラーハンドリング

Promiseの中でもエラー処理が可能です。

/* Promiseにおけるエラーハンドリングの例 */
let myPromise = new Promise((resolve, reject) => {
    setTimeout(() => {
        reject("Promiseが拒否されました");
    }, 1000);
});

myPromise
    .then(result => {
        console.log(result);
    })
    .catch(error => {
        console.error("エラー:", error);
    });

まとめ

JavaScriptのエラーハンドリングを理解することで、堅牢なアプリケーションを作成できるようになります。
