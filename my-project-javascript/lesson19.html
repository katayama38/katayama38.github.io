JavaScriptのテストとデバッグの理解

テストの定義

テストは、プログラムが期待通りに動作することを確認するためのプロセスです。

/* 簡単なテストの例 */
function add(a, b) {
    return a + b;
}
console.assert(add(2, 3) === 5, 'テスト失敗: 2 + 3 は 5 でなければなりません。');

ユニットテスト

ユニットテストは、個々の関数やモジュールを単体でテストする手法です。

/* Jestを使ったユニットテストの例 */
test('2 + 3 は 5 である', () => {
    expect(add(2, 3)).toBe(5);
});

デバッグの定義

デバッグは、プログラム内のバグを特定し修正するプロセスです。

/* デバッグの例 */
const numbers = [1, 2, 3];
numbers.forEach(num => {
    console.log(num); // 各数値を表示
});

コンソールの使用

ブラウザのコンソールを使用して、変数の値やエラーメッセージを確認できます。

/* コンソールの使用例 */
console.log('デバッグ情報');
console.error('エラーが発生しました！');

ブレークポイントの設定

開発者ツールを使って、特定の行でコードの実行を一時停止し、状態を確認できます。

/* ブレークポイントの設定はコード内ではなく、ブラウザの開発者ツールで行います。 */
function multiply(a, b) {
    return a * b; // この行にブレークポイントを設定
}

エラーハンドリング

try-catch文を使用して、エラーが発生する可能性のあるコードを安全に実行できます。

/* エラーハンドリングの例 */
try {
    const result = riskyFunction();
} catch (error) {
    console.error('エラー:', error);
}

まとめ

JavaScriptのテストとデバッグを理解することで、より信頼性の高いコードを作成することができます。
