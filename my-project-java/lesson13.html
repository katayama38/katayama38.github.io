Javaの例外処理

例外処理の概要

例外処理は、プログラムの実行中に発生するエラーを管理するためのメカニズムです。これにより、プログラムがクラッシュせずに適切にエラーを処理できます。

基本的な例外処理の構文

Javaでは、例外処理を行うために`try`、`catch`、`finally`ブロックを使用します。

/* 例外処理の基本例 */
public class ExceptionExample {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // ゼロ除算を試みる
        } catch (ArithmeticException e) {
            System.out.println("エラー: " + e.getMessage());
        } finally {
            System.out.println("このメッセージは必ず表示されます。");
        }
    }
}

例外の種類

Javaには、チェック例外と非チェック例外の2種類の例外があります。チェック例外はコンパイル時に処理を強制され、非チェック例外はランタイム時に発生します。

カスタム例外の作成

独自の例外を作成することも可能です。これは、特定のエラー条件を管理するのに役立ちます。

/* カスタム例外の例 */
class MyException extends Exception {
    public MyException(String message) {
        super(message);
    }
}

public class CustomExceptionExample {
    public static void main(String[] args) {
        try {
            throw new MyException("カスタム例外が発生しました");
        } catch (MyException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}

例外の再スロー

キャッチした例外を再スローして、呼び出し元にエラーを伝えることもできます。

/* 例外の再スローの例 */
public class RethrowExample {
    public static void main(String[] args) {
        try {
            methodThatThrows();
        } catch (Exception e) {
            System.out.println("例外をキャッチしました: " + e.getMessage());
        }
    }

    static void methodThatThrows() throws Exception {
        throw new Exception("何かのエラーが発生しました");
    }
}

まとめ

例外処理を正しく使うことで、プログラムの信頼性が向上し、エラー発生時の挙動を制御できます。
