Javaのコレクションフレームワーク

コレクションフレームワークの概要

Javaのコレクションフレームワークは、データの集合を管理するためのクラスとインターフェイスの集合です。リスト、セット、マップなどのデータ構造を提供します。

Listインターフェイスの例

リストは、順序付きの要素の集合で、重複を許可します。

/* Listインターフェイスの例 */
import java.util.ArrayList;
import java.util.List;

public class ListExample {
    public static void main(String[] args) {
        List fruits = new ArrayList<>();
        fruits.add("リンゴ");
        fruits.add("バナナ");
        fruits.add("オレンジ");

        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}

Setインターフェイスの例

セットは、順序を持たない要素の集合で、重複を許可しません。

/* Setインターフェイスの例 */
import java.util.HashSet;
import java.util.Set;

public class SetExample {
    public static void main(String[] args) {
        Set animals = new HashSet<>();
        animals.add("犬");
        animals.add("猫");
        animals.add("犬"); // 重複は無視される

        for (String animal : animals) {
            System.out.println(animal);
        }
    }
}

Mapインターフェイスの例

マップは、キーと値のペアを管理するためのコレクションです。キーはユニークである必要があります。

/* Mapインターフェイスの例 */
import java.util.HashMap;
import java.util.Map;

public class MapExample {
    public static void main(String[] args) {
        Map ages = new HashMap<>();
        ages.put("太郎", 25);
        ages.put("花子", 30);
        ages.put("次郎", 22);

        for (Map.Entry entry : ages.entrySet()) {
            System.out.println(entry.getKey() + "の年齢は " + entry.getValue() + " 歳です。");
        }
    }
}

コレクションの利点

コレクションフレームワークを使用することで、データの管理が容易になり、アルゴリズムを適用するための標準的なインターフェイスが提供されます。

まとめ

Javaのコレクションフレームワークを活用することで、データ構造の操作を簡素化し、効率的なプログラムを作成できます。
