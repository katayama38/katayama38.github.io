Javaのインターフェイスと抽象クラス

インターフェイスの概要

インターフェイスは、クラスが実装すべきメソッドの集合を定義します。インターフェイスは多重継承が可能です。

インターフェイスの例

インターフェイスの定義とその実装を示します。

/* インターフェイスの例 */
interface Animal {
    void sound(); // メソッドの宣言
}

class Cat implements Animal {
    @Override
    public void sound() {
        System.out.println("猫が鳴いています");
    }
}

class Dog implements Animal {
    @Override
    public void sound() {
        System.out.println("犬が吠えています");
    }
}

// 使用例
Animal myCat = new Cat();
myCat.sound(); // 猫の鳴き声

Animal myDog = new Dog();
myDog.sound(); // 犬の吠え声

抽象クラスの概要

抽象クラスは、他のクラスが拡張するための基盤を提供します。抽象メソッドを持ち、具体的な実装はサブクラスで行います。

抽象クラスの例

抽象クラスとそのサブクラスの実装を示します。

/* 抽象クラスの例 */
abstract class Animal {
    abstract void sound(); // 抽象メソッド
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("猫が鳴いています");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("犬が吠えています");
    }
}

// 使用例
Animal myAnimal = new Cat();
myAnimal.sound(); // 猫の鳴き声

myAnimal = new Dog();
myAnimal.sound(); // 犬の吠え声

インターフェイスと抽象クラスの違い

インターフェイスは複数のクラスに実装可能で、抽象クラスは単一のクラスからのみ継承できます。インターフェイスはフィールドを持たず、抽象クラスはフィールドを持つことができます。

まとめ

インターフェイスと抽象クラスを理解することで、柔軟で再利用性の高いコードを設計できます。
